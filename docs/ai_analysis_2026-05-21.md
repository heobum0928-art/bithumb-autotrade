## 1. 필터 품질 평가

### 🔴 **심각한 문제: 필터가 역할을 못하고 있음**

#### A. RSI 범위 필터 - **매우 우수** ✅
```
RSI > 95: -8.35% (5분), UP RATIO 0% → 완벽한 차단
RSI > 92: -3.13% (5분), UP RATIO 12% → 양호
```
**결론**: RSI 범위 필터는 정상 작동. 현재 `45~90` 설정은 적절함.

#### B. BB (볼린저밴드) 필터 - **모순적이고 위험** 🚨
```
BB 1.47: -0.74% (5분), UP 11% → 차단해야 함
BB 1.40: +2.56% (5분), UP 71% → 차단하면 안 됨
BB 1.38: +3.91% (5분), UP 79% → 가장 좋은 신호
```

**핵심 발견**: 
- **과열(1.47) 상태가 하락신호** → 현재 `bb_limit: 1.3`은 **너무 높음**
- **1.38~1.40 대역이 상승신호** → 오버부이드 기준이 잘못됨
- **결론**: 현재 필터가 손절 신호를 놓치고 있음

#### C. 호가불균형 필터 - **약함** ⚠️
```
200건 중 UP 34% → 차단 효율 저조
평균 -1.13% (5분) → 큰 손실은 아니지만 불필요 진입
```

#### D. 체결강도 필터 - **우수** ✅
```
103건 중 UP 26% → 잘 작동
평균 -1.72% (5분) → 효과적 차단
```

## 2. 펌핑 패턴 인사이트

### 📊 **펌핑 이벤트의 심각한 함정**

```
총 펌핑 신호: 367건
평균 펌핑: +4.98% (매력적으로 보임)

하지만 현실:
- 2% 이상 급락(pullback): 86% (317건)
- 반등: 39% (123건)
- 실제 5분 평균 수익률: -2.39% ← 매우 위험
```

### **핵심 문제**
1. **높은 변동성 함정**: 초기 펌핑 후 즉시 급락
2. **평균 4.98%의 펌프는 대부분 함정** 
   - `pullback_rate 86%` = 진입 후 86% 확률로 손절
   - `avg_5m_return -2.39%` = 실제 수익 불가능

### **진입 신호의 허상성**
- 볼륨 상승(avg_vol_mult 25.1배)이 역설적으로 **폭락 신호**
- 현재 필터들이 이를 제대로 걸러내지 못함

## 3. 눌림목 전략 실현 가능성

### 📈 **현재 데이터로는 거의 불가능** ❌

#### 원인 분석:

**A. 포지션 홀드 불가능**
```
평균 홀드 시간: 100~600초 (최대 10분)
- 눌림목 형성: 최소 5~15분 필요
- 현재: "빠른 손절"이 기본 동작

예시: META 코인
- Hold: 38~153초 (2~3분)
- 손절율: 매우 높음
- 눌림목 확인 불가
```

**B. stop_pct -1.5%의 문제점**
```
최근 7일 거래:
- 초기 손절 사유 67% (9/14건)
- 진입 후 2~8분 내 급락으로 손절
- 눌림목이 형성되기도 전에 청산

META의 경우만 예외 (win_rate 높음):
- 조기 트레일링으로 작은 수익 확정
- 눌림목이 아닌 "즉시 상승" 패턴만 수익
```

**결론**: 
> **현재 설정으로는 눌림목 전략이 아닌 "펌프앤덤프 회피" 전략**
> 눌림목으로 전환하려면 근본적 재설계 필요

## 4. 즉시 개선 가능한 조정안

### 🔧 **우선순위 1: 볼린저밴드 리캘리브레이션** (즉시 적용)

```python
# 현재 설정
"bb_limit": 1.3,  ← 너무 높음

# 개선안
"bb_limit": 1.38,  # 1.38~1.40이 상승 신호 (UP 79~71%)
"bb_strict_mode": True,  # 1.40 이상은 완전 차단
```

**근거**:
- BB 1.38: avg +3.91% (5분), UP 79%
- BB 1.47: avg -0.74% (5분), UP 11%
- **변경 효과**: 거짓 신호 약 40% 감소 예상
