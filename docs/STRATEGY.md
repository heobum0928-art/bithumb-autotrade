## 🔧 운영 프로토콜 (의사결정 구조 — 2026-06-13 확정)

**논의 = Claude(에이전트) · 실행 = 봇(룰) · 실거래 전환 = 사용자 승인**

- **진입/청산 실행**: 100% 봇 자동(룰 기반). 사람이 매 진입을 승인하지 않음 — 단타는 초~분 단위라 텔레그램 왕복·부재(수면)시간에 신호가 유실됨(CI가 1902건 중 1894건 무응답으로 멈춘 전례).
- **진입 채택 기준**: 승률 임계 ❌ → **기대값(비용 0.16% 차감 후 평균수익 > 0)** ✅. "승률 50% 넘으면 진입"은 트레일링(승률47%·수익최고) 같은 비대칭 전략을 버리므로 금지. 운영자 철학 "승률보다 수익, 많이 먹고 빠지기"와 일치.
- **전략·규칙 변경(논의)**: Claude 에이전트 팀 논의 → 기대값 검증 → **봇 코드화** → 반영.
  - **코드화 = 봇이 모든 케이스(코인·시점·예외상황)마다 규칙을 빠짐없이 확인하도록 구현.** 클로드가 제안한 조건을 봇이 케이스별로 다 체크해 진입/청산 판단. 구현 후 [[feedback_edgecase_loop]] 엣지케이스 7종 점검 필수(예: 2026-06-13 WS구독 버그 = "제외 코인 보유" 케이스 누락).
  - 모의(dry-run): **사후통보로 대체**(텔레그램/로그로 "규칙 이렇게 바꿈" 통지). 수면 시간에도 검증이 멈추지 않게.
  - 실거래: **사전승인 유지**(사용자 돈이 나가는 결정은 반드시 사용자 검토 후).
- **확률·통계(%)**: 진입 트리거가 아니라 사후통보·검증지표·실거래 전환 판단 근거로 사용.
- 근거: 10팀·8팀·6팀 에이전트 논의 수렴. Claude의 직접 매수추천은 정책상 거부되므로 "규칙(룰)"로 우회.
- **자율 리서치 권한 (2026-06-18 사용자 위임)**: Claude가 ①유망전략 발견 → ②백테스트(0원) → ③에이전트팀 red-team(과최적화 점검) → ④수렴 시 dry-run 배포해 데이터 축적 — **여기까지 매번 승인 없이 자율.** 단 **실거래 전환(진짜 돈)은 게이트 통과 후에도 반드시 사용자 승인.** 가드레일: 폐기 13개 메커니즘 새증거 없이 재시도 금지 / 후보당 공정 백테스트 1회(tweak-retry 금지) / 에이전트는 데이터 검증 시만 소집. 트리거 주로 = 장세전환(강세장 RT재가동)·새데이터 착수. daily_strategy_report.py가 기회 포착.
- ⚠️ **자만 경계 (2026-06-13)**: **프로세스 우위 ≠ 수익 우위.** 검증이 빡빡하다는 건 "과최적화 가짜수익에 안 속는다"(손실 방어)일 뿐, "더 번다"가 아니다. 현재 누적 -49만원, 모의도 통계 미달(VB t1.50/RT t0.49, 둘 다 No-Go). **진짜 판정은 게이트 통과 + 실거래 수익뿐. 그 전까진 아무것도 증명된 게 없다.** 외부 사례(유튜브 봇 등)와 "수익"으로 비교 자체가 불가(양쪽 다 미입증).

**RT 보조검증 (2026-06-16, 백테스트 90일·42코인·트레일3%·TEST구간)**:
- **비용 스트레스**(scripts/_cost_stress_test.py): RT 엣지가 실전비용에 안 무너짐. 비용 0.16%→t2.44, 0.30%→t1.91, 0.50%→t1.15, 비용후평균은 0.5%까지 계속 플러스(+0.30%). 쿠폰0.04% 유지가 생존 핵심.
- **장세필터**(scripts/_regime_filter_test.py): "BTC가 N봉SMA 위일 때만 진입". 짧은 필터(N=12,1h)만 도움(t1.83→1.94, MDD17→12%), 긴 필터(12h·24h)는 해로움. **보조개선 수준, 구원투수 아님.** ⚠️이 스크립트 초판은 비용 이중차감 버그 있었음 — 정정값은 _cost_stress_test 기준.
- **핵심 발견**: live 모의 +2.72%(17건)는 강세장+소표본 뻥튀기. 176건 백테스트 진짜 엣지는 비용후 **+0.5%대**(얇지만 비용 견딤). t≥2를 n=30서 못 넘는 건 "무에지"가 아니라 "변동성 큰 얇은 엣지엔 30이 부족"(필요 ~35건). → **중단규칙 후보: RT 40건서 비용후>0&t≥2면 소액실거래, 아니면 단타 정리.** (사전등록 대기)

**트레일 청산 튜닝 (2026-06-17, scripts/_trail_tune_test.py, TEST 29코인)**: 고정3% 트레일이 잡코인 변동성에 헐거워 고점수익 게워냄 관찰 → 6방식 비교.
