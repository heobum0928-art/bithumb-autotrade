## 필터 구조 (현재)

1. **BTC 하락 필터** — BTC -1.5% 이상 시 진입 차단
2. **거래량 지속성 필터** — 이전 30초도 최소 거래 활성 확인
3. **RSI 필터** — 45~90 구간만 진입 (과열 90↑ / 침체 45↓ 차단)
4. **MACD 필터** — 하락 추세 차단 (데이터 부족 시 통과)

8. **체결강도 필터** — 매수틱 비율 60% 이상
9. **블랙리스트 필터** — COIN_BLACKLIST 코인 차단
10. **시간대 필터** — DEAD_HOURS_KST 차단
11. **30초 확인 딜레이** (즉시진입 코인만) — 가격 유지 확인

## 코인 분류 (2026-05-25 기준)

### COIN_BLACKLIST (진입 금지)
```python
{"PROS", "UP", "RSS3", "AQT", "BMT", "FCT2"}
```
- 근거: pump_log 반등 0% (5건 이상) 또는 실거래 손실 누적
- **제거된 코인**: SUNDOG(+18,872원), PUFFER(+7,147원), DAO(+14,096원), XION(+13,483원)
  → 실거래에서 수익 확인 → 블랙리스트가 오히려 수익 차단하고 있었음

### IMMEDIATE_ENTRY_COINS (즉시 진입)
```python
{"META", "WNCG"}
```
- META: 4건 100% 승률, +1,491원 실거래 검증
- WNCG: pump_log 강반등 패턴 (+14% 시그널 확인)
- 나머지: PULLBACK_ENABLED=True → 눌림목 대기

### DEAD_HOURS_KST (진입 금지 시간대)
```python
{6, 7, 11, 12, 13, 14, 15}
```
- 근거: pump_log 시간대별 반등률 분석 — 이 시간대 반등률 0~15%
- **포함하지 않은 시간**: 01~05시 → pump_log 01시 반등률 80.9% → 차단 불필요

## 핵심 발견 (데이터 분석 결과)

### 1. OB_BID_RATIO=1.5 는 구조적 버그였음 (2026-05-25 발견)
- 펌핑 중에는 매도 주문이 몰려 bid/ask 비율이 구조적으로 1.0 미만
- 1.5 기준으로 모든 신호가 차단되어 봇이 0건 진입
- → 0.0으로 비활성화 후 정상 진입 재개

### 2. 블랙리스트가 역효과였음 (2026-05-25 발견)
- 블랙리스트 코인 실거래 합계: +43,711원 (이익!)
- 비블랙리스트 코인 실거래 합계: -107,676원 (손실)
- pump_log 반등률 낮다고 차단한 코인들이 실제로는 수익 코인이었음
- → 실거래 PnL 기반으로 블랙리스트 재설계

### 3. 눌림목 황금구간은 -3~5%
- -3~5% 낙폭 구간 반등률: 61.5%
- -7% 낙폭 구간 반등률: 20%
- → PULLBACK_TARGET_PCT = -0.035 (-3.5%)

### 4. 홀딩 타임아웃이 최대 손실 원인
- "5분 내 목표 미달" = 19건, -22,348원 (전체 손실의 35%)
- 5분 안에 반등 안 오면 손실로 청산되는 패턴
- → PULLBACK_WAIT_SEC=300 (5분) 적절한지 재검토 필요

### 5. 즉시진입 전략의 실제 성과
- 조기 진입 후 즉시 반등한 코인: SUNDOG, DAO, XION, BSV, PUFFER
- 공통점: 신규 상장 또는 강한 내러티브 보유
- 즉시진입 코인 분류의 기준은 pump_log peak_at_sec (펌프 속도)
