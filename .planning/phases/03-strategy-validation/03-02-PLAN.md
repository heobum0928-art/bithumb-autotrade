    그리드 상수 (Discretion — 현 상수 진입 -7/TP +5/SL -3 중심,
    파라미터당 3~4단계, D-10):
    ```python
    import itertools
    ENTRY_GRID = (0.05, 0.07, 0.10)          # 진입 눌림 -5 / -7 / -10%
    TP_GRID    = (0.03, 0.05, 0.07, 0.10)    # 익절 +3 / +5 / +7 / +10%
    SL_GRID    = (0.02, 0.03, 0.05)          # 손절 -2 / -3 / -5%
    GRID_SLIPPAGE = 0.01                     # D-02: 그리드도 슬리피지 1% 기준선
    # 총 3 × 4 × 3 = 36 조합
    ```

    `grid_search(train_events: list[dict], db_path, slippage: float = GRID_SLIPPAGE) -> list[dict]`:
    - `itertools.product(ENTRY_GRID, TP_GRID, SL_GRID)`로 36조합 순회
    - 각 조합 → `strat = Strategy(entry_drop_pct=entry, tp_pct=tp, sl_pct=sl)`
    - 각 train 이벤트 → `ticks = get_ticks(ev["id"])`,
      `t = simulate_event(ticks, slippage, strat)`; `t is not None`이면
      `t["coin"] = ev["coin"]` 붙이고 trades에 추가
    - `m = compute_metrics(trades)` 후 결과 행:
      `{"entry": entry, "tp": tp, "sl": sl, "event_count": len(train_events), **m}`
      (compute_metrics가 빈 trades에 `{"count": 0}`만 반환하므로
      `m.get("ev", ...)` 형태로 안전 접근할 것)
    - 정렬 (D-11 + RESEARCH Open Question 3): EV 내림차순 1차,
      승률 내림차순 2차, MDD 오름차순 3차:
      `results.sort(key=lambda r: (-r.get("ev", -1e9), -r.get("win_rate", 0.0), r.get("mdd", 1e9)))`
    - 정렬된 results 리스트 반환. `results[0]`가 train 최고 EV 조합.

    **규율(D-11):** `grid_search`는 `EventSplit.train`만 인자로 받아야
    한다 — 호출부에서 `split.train`을 넘기고, 절대 전체 이벤트 리스트나
    `split.test`를 넘기지 않는다. 함수 docstring에 이 계약을 명시.
